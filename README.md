## <img src="https://media.giphy.com/media/VgCDAzcKvsR6OM0uWg/giphy.gif" width="35"> &nbsp; How I Think

```diff
@@ ─────────── My Builder Philosophy ─────────── @@

- Build for resumes
+ Build for real people with real problems

- Jump straight to solutions
+ Diagnose the root failure first

- "What features should I add?"
+ "What is actually breaking, and WHY?"

- Optimize for looking productive
+ Optimize for measurable real-world impact

- Ship when it's perfect
+ Ship when it's validated — then iterate

! I care more about WHY systems fail than HOW to code them.
! Speed of right iteration > perfection of first draft.
! If I can't name exactly who benefits and how → I don't ship it.
```
